## Vector DB Use Case

Traditional keyword search is insufficient for legal documents because it cannot understand meaning. It only matches exact words.

Vector databases convert text into embeddings, capturing semantic meaning. This allows similarity search based on context, not just keywords.

For example, "termination clause" and "contract ending terms" are different words but similar meaning. Vector DB can match them.

Thus:
- Better search accuracy
- Semantic understanding
- Faster retrieval

Vector DB is essential for this use case.
